使用github+hexo十分钟搭建专属博客

2021-07-17
博客
  • Hexo
有 N 人看过

简介: 本文通过简单方式可以让你在十分钟内完成 github + hexo 博客的部署；写下此文的时候 gitee 的 page 服务在维护，如果使用 gitee + hexo 部署，国内访问速度会大大提升。

前提

默认读者是一个开发者，能够了解基础的软件网站知识。

本文使用环境:

  • macOS 10.15.7

环境准备

1.点击node官网地址下载 node.js

# node -v
v12.13.0

2.点击git官网地址下载git版本管理工具

# git --version
git version 2.15.0

3.创建 github 仓库

1.Repository name 填写 Owner.github.io
比如我的 Owner 叫：dm，那么我的 Repository name 只需要填写: dm.github.io 即可
2.点击下方绿色的 Create repository

本地搭建

1.在电脑本机找一个空的文件夹，比如我的是: ~/Desktop/blog，cd 进入该文件夹执行:

npm install hexo-cli -g

用来全局安装 hexo 博客

2.安装完 hexo-cli 之后，使用 hexo 初始化此文件夹

hexo init

3.初始化完成之后按照顺序执行以下三个命令

hexo clean # 清空已有hexo网站文件
hexo generate # 依据网页文本与新的CSS样式生成新网站文件
hexo server # 启动本地服务器，可以在localhost:4000查看网站修改效果

可以简化为:

hexo clean && hexo g && hexo s

出现 “ Hexo is running at http://localhost:4000 . Press Ctrl+C to stop” 时，证明 Hexo 已经启动，可以打开浏览器进行访问

为博客更换主题

1.访问 https://hexo.io/themes/，寻找喜欢的主题。

到博客项目下面的 theme 文件夹下面执行 git clone 命令下载喜欢的主题代码。
我的路径是: ~/Desktop/blog/theme，我的博客主题是Aomori

git clone https://github.com/lh1me/hexo-theme-aomori

执行完这个命令后，能够看到，多了一个 hexo-theme-aomori-master 文件夹，这个就是 Aomori 主题。

2.使用该主题
回到当前所处的文件夹 theme 的上一级(即我们执行 hexo init 命令时所在的目录)，可以看到有个文件叫 _config.yml。
把这个文件中 theme: landscape 配置项改为 theme: hexo-theme-aomori-master , 一定要注意此配置文件是yaml格式的，所以冒号后面的空格不可以少哦。

这个文件的文档可以参照 https://hexo.io/zh-cn/docs/configuration。

保存修改之后，在当前目录进行命令三连。

hexo clean && hexo g && hexo s

打开http://localhost:4000就可以看到主题应用成功了。

部署

1.复制刚刚在 github 创建的仓库HTTPS地址。

https://github.com/lh1me/hexo-theme-aomori.git

2.同样修改_config.yml 最后几行：

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: https://github.com/lh1me/hexo-theme-aomori.git
  branch: master

这里的目的是：让 hexo 知道把博客推到哪个仓库的哪个分支上。

3.安装deployer

npm install --save hexo-deployer-git

4.三连部署

hexo clean && hexo g && hexo d

这次最后一个字母变成了 d， 代表的是部署。出现 INFO Deploy done: git 代表部署成功。

pages 服务

到 github 仓库中就可以看到刚刚部署推送的代码了。

1.找到仓库的 settings 按钮入口，进入之后点击左侧 pages 就可以看到你的博客域名了。

点击进去就可以看到博客的样子了。
